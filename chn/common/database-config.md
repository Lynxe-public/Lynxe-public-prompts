# 数据库配置

## 功能说明

本系统内置了数据库支持能力，MySQL或本地H2都可以。

有了数据库，就会比较方便做一些需要做大量数据处理的场景，然后数据库本身也支持高效导出为Excel，所以比较适合做核心共享存储。

同时，因为LLM会更简单地处理建库建表等逻辑，所以基本上只要把库配置好，当成各类场景的临时结构化存储，效果都不错。

## 具体配置步骤

**第1步** 打开右上角配置
- 点击系统右上角的配置图标

**第2步** 点击数据库配置
- 在配置菜单中选择"数据库配置"选项

**第3步** 依次输入配置信息
- **数据库名称**: 对应MySQL/H2的schema名称
- **类型**: 选择数据库类型（MySQL或H2）
- **URL**: 数据库连接的套接字地址
- **密码**: 数据库访问密码

**第4步** 测试连接
- 点击"测试连接"按钮
- 确认连接成功

**第5步** 保存配置
- 测试通过后，点击"保存"按钮
- 配置完成

## 预期结果

- 成功配置数据库连接
- 系统可以正常访问数据库
- 支持后续的数据库操作（建表、查询、写入等）

## 技术要点

- 支持MySQL和H2两种数据库类型
- 数据库作为核心共享存储
- 支持数据导出为Excel格式
- LLM可以自动处理建库建表逻辑
- 适合作为各类场景的临时结构化存储
