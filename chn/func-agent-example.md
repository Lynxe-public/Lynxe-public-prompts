# 函数化Agent示例

## 功能说明

这是一个展示函数化Agent（Function-Agent）设计理念的示例，演示了如何通过函数组合和路由机制，实现智能问题分类和专家系统。

你可以从这里了解如何创建多个专业化的Agent函数，如何实现函数之间的组合调用，以及如何通过路由Agent将问题分配给合适的专家处理。

这个例子中的 技术专家，数据库专家和业务处理专家，对应三个子agent， 他们拥有完全独立的记忆资源，并且可以并行使用，这是目前我们实践下来最好的使用sub-agent的最佳实践。

## 使用流程

**版本要求**: 4.8.6（含）以上

**第1步** 打开Lynxe系统

**第2步** 导入数据
- 导入 [prompt-script/func-agent-example.json](prompt-script/func-agent-example.json) 文件中的计划模板
- 系统会自动识别并加载所有函数Agent

**第3步** 理解函数结构
- **技术专家处理问题**: 专门处理技术类问题的Agent函数
- **数据库专家处理问题**: 专门处理数据库类问题的Agent函数
- **业务专家处理问题**: 专门处理业务类问题的Agent函数
- **主入口-路由方法**: 根据问题类型自动路由到相应专家的主入口函数 这是最核心的启动函数，你也需要确保他的正确的引用了其他三个func—agent，这应该是自动的，但有可能因为版本不同而存在被错误忽略的问题

**第4步** 执行路由函数
- 运行"主入口-路由方法"函数
- 输入用户需求或问题
- 系统会自动判断问题类型并路由到相应的专家函数
- 专家函数处理后会返回相应的答案


## 预期结果

- 成功导入4个函数Agent：路由函数和3个专家函数
- 路由函数能够正确识别问题类型（技术/数据库/业务）
- 问题被自动路由到对应的专家函数进行处理
- 各专家函数返回符合其专业领域的答案
- 支持函数组合和复用，体现"函数是第一公民"的设计理念

## 技术要点

- Function-Agent能力：智能函数组合和自动任务分解，这也是整个系统最关键的特征之一
- 函数组合和复用：支持将复杂任务分解为可复用的工具函数
- 路由机制：根据问题类型自动选择最合适的处理函数
- 函数发布：所有函数都可以作为独立的工具被调用
- 参数传递：支持函数间的参数传递和结果返回
